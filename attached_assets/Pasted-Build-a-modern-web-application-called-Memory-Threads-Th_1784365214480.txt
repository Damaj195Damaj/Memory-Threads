Build a modern web application called "Memory Threads".

The vision:
Memory Threads is an AI-powered memory engine for a user's files. Instead of searching by filename, users search by memory, context, people, events, descriptions, or conversations.

Examples:
- "The PDF my professor sent before the midterm."
- "The screenshot with the purple error."
- "The presentation mentioning AI agents."
- "What did John tell me about authentication?"
- "What happened last Tuesday?"

This is NOT a chatbot. The AI should reason over a persistent knowledge base built from uploaded files.

CORE FEATURES
---------------------------------

1. Upload files

Support:

- PDF
- DOCX
- TXT
- Images
- CSV
- Markdown

Future-ready architecture so audio/video can be added later.

---------------------------------

2. Automatic Processing

For every uploaded file:

- Extract text
- Generate AI summary
- Detect people
- Detect organizations
- Detect dates
- Detect locations
- Detect tasks
- Detect topics
- Generate embeddings
- Save metadata
- Save vector
- Build relationships

---------------------------------

3. Memory Cards

Each file becomes a beautiful Memory Card containing:

- title
- preview
- summary
- tags
- people
- dates
- organizations
- confidence
- related memories

---------------------------------

4. AI Search

Search bar accepts natural language.

Examples:

"The PDF about climate change."

"The image with three people."

"The invoice due next week."

"My contract with Apple."

Return ranked memories.

Each result shows WHY it matched.

---------------------------------

5. Ask Memory

Users can ask questions like:

"What should I do next week?"

"What documents mention React?"

"What files involve John?"

"What happened before the launch?"

The assistant answers using retrieved memories.

Every answer must cite its sources.
